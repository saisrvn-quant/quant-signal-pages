FRAMEWORK

Rulebook last rebuilt: 2026-05-15T10:01:20 (UTC) · Rebuild cadence: 1st & 15th of each month

Method overview

Every bar is classified into a (zone, ema_stack, macd_h) triplet. The rulebook groups the historical record by triplet, computes forward returns at multiple windows, and keeps only triplets where returns are consistently positive (or consistently negative for shorts) across all windows. The current bar's triplet is then matched against the rulebook each run.

L0–L5 indicator hierarchy (37 indicators)

  • L0 Pivot/CPR: zone, dist_P, cpr_width, cpr_dir, cpr_vrg
  • L1 Trend: ema_stack (EMA9/18/27), vwap_pos, lr_slope, ema_cross
  • L2 Momentum: rsi, rsi_div, macd_h (NEG_GROW/NEG_SHRK/POS_GROW/POS_SHRK), roc, smi
  • L3 Structure: bb_width, bb_sq, atrp, adx, macd_h_chg
  • L4 Regime: regime_label (ADX+EMA stack), vol_spike_flag (vol-of-vol expansion)
  • L5 Volatility: atr_of_atr, ewma_vol, roll_std

Pivot construction (L0)

  • 2HR bars: previous-WEEK high/low/close → weekly pivots → zone bins
  • Daily bars: previous-MONTH high/low/close → monthly pivots → zone bins
  • Zones: BELOW_S3, S3, S3_S2, S2, S2_S1, S1, BC (below pivot inside CPR), TC (above pivot inside CPR), R1, R1_R2, R2, R2_R3, R3, ABOVE_R3

Forward-return windows (consistency gate)

  • 2HR setups: bars 3, 6, 9 — must be positive (long) or negative (short) at ALL three
  • Daily setups: days 5, 10, 20 — must be positive (long) or negative (short) at ALL three

What the rulebook is NOT

  • Not a backtested strategy — these are descriptive groupby statistics, in-sample
  • No transaction costs / slippage modelled — close-to-close arithmetic returns
  • No statistical significance tests on win rates — interpret n carefully
  • VOL_SPIKE_FLAG is a vol-of-vol threshold, not real change-point detection
  • EWMA_VOL is exponentially-weighted squared returns, not fitted GARCH

CHECKLIST

Run every step before any trade. Skipping the daily-alignment check is the most common cause of losing trades.

Step 1 — Asset character check

  • NG: Check conflict zones first (BELOW_S3, R2, S1, S3 = 0%, stop here)
  • NIFTY: Check conflict (S2 daily-bear → 0% for longs)
  • NQ, TSX, GC: No conflicts — proceed

Step 2 — Zone classification

  • S-zones / BC / TC / BELOW_S3 → TYPE A CONTRA (full base alloc)
  • R-zones / ABOVE_R3 → TYPE B TREND (base × 0.6)
  • NG at R2_R3 / R3 / ABOVE_R3 → CONSIDER SHORT not long

Step 3 — Layer 1: primary signal

  • Does the current (zone, ema_stack, macd_h) appear in the top-setups table on its asset's tab?
  • Is it positive at all forward windows (3+6+9 for 2HR, 5+10+20 for daily)?
  • If no → DO NOT TRADE (no exceptions)

Step 4 — Layer 2: ADX regime

  • Cross-reference current ADX bucket against asset-specific avoid list
  • NQ avoids: S2_S1+RANGE, S3_S2+STRONG_TREND, ABOVE_R3+RANGE
  • NG avoids: ABOVE_R3+STRONG_TREND, S1+STRONG, R1_R2+STRONG, R2_R3+TREND
  • TSX avoids: S3_S2+STRONG, TC+STRONG, ABOVE_R3+STRONG, BC+RANGE
  • GC avoids: S3_S2+TREND, R2+STRONG_TREND
  • NIFTY avoids: TC+STRONG_TREND, R2_R3+TREND, BC+STRONG, S1+STRONG
  • Conflict → reduce alloc 30%

Step 5 — Layer 3: RSI timing

  • Match current RSI state against the cheat sheet on the FRAMEWORK tab
  • Mismatch → wait 1 bar before entering

Step 6 — Daily alignment

  • Look at the asset's tab → "Daily bull zones" line
  • If current 2HR zone is in bull_d list AND going long → alloc × 1.5 (cap 90%)
  • If current 2HR zone is in bear_d list AND going long → 0% (skip)
  • Neither → alloc × 1.0

Step 7 — VOL_SPIKE_FLAG check

  • If flag = 1 (vol-of-vol expansion firing on current bar) → subtract 20pp from alloc

Step 8 — Final alloc calculation

  1. Base alloc from zone
  2. × 0.6 if TYPE B
  3. × 1.5 if daily aligned (cap 90%)
  4. − 20pp if VOL_SPIKE_FLAG = 1
  5. + 5pp if MACD = NEG_SHRK at entry
  6. Cap at 30% if watchlist
  7. Hard cap 90%

Step 9 — Position size

  • NQ: 25–30% of Kelly fraction × alloc%
  • NG: 20% of Kelly × alloc% (high vol, smaller sizing)
  • TSX: 25% of Kelly × alloc%
  • GC: 25% of Kelly × alloc%
  • NIFTY: 25% of Kelly × alloc%

Step 10 — Stops & targets

  • Stop = p25 of that specific combo (asset tab shows it)
  • Target = p75 of that combo
  • Time stop: 9 bars (2HR) or 20 bars (daily) — exit regardless

Step 11 — Re-check at every new bar

  • If direction changes → exit and re-enter only on next valid signal
  • If alloc drops by ≥10pp → reduce position correspondingly
  • If setup_id rotates to different combo → exit, evaluate new combo

Step 12 — Hold-max enforcement

  • 2HR setups: exit by bar 9 even if green
  • Daily setups: exit by day 20 even if green
  • Edge decays beyond these windows — letting it run is gambling, not strategy
